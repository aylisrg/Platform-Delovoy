# PRD: Мега-лендинг — видео на Hero + отзывы с Яндекс Карт

## Проблема

Текущий лендинг бизнес-парка "Деловой" — статичная страница без визуального вау-эффекта. Hero секция — чистый текст на чёрном фоне. Нет социального доказательства: упоминание "300+ отзывов" есть, но сами отзывы не показаны. Посетитель не может оценить атмосферу парка без фото/видео и не видит реальных мнений клиентов.

## Решение

1. **Видео-фон на Hero** — фоновое видео на весь первый экран (loop, muted, autoplay). Поверх видео — затемнение + существующий текст, badge, CTA кнопки и статистика. Видео загружается пользователем в папку `public/media/`.

2. **Секция отзывов с Яндекс Карт** — новая секция между "Преимущества" и "Контакты". Серверный парсинг отзывов с Яндекс Карт через API route (`GET /api/reviews`), кэширование в памяти/Redis. Отображение в виде карусели с именем, рейтингом, текстом отзыва и датой.

## Целевая аудитория

- **USER** — потенциальный арендатор или посетитель парка. Видит премиальную подачу, читает реальные отзывы, принимает решение обратиться.

## User Stories

### US-1: Видео-фон на главном экране

- **Как** посетитель сайта (USER)
- **Я хочу** видеть видео бизнес-парка на первом экране
- **Чтобы** сразу почувствовать атмосферу и уровень парка

**Acceptance Criteria:**
- [ ] AC-1: На Hero секции проигрывается фоновое видео (autoplay, loop, muted) на весь экран
- [ ] AC-2: Поверх видео — полупрозрачный тёмный overlay для читаемости текста
- [ ] AC-3: Текст, badge, CTA кнопки и статистика отображаются поверх видео без изменения контента
- [ ] AC-4: Видео загружается из `public/media/hero.mp4` (путь конфигурируемый)
- [ ] AC-5: Если видео не загружено / ошибка — fallback на текущий чёрный фон (graceful degradation)
- [ ] AC-6: На мобильных устройствах видео отключается (показывается постер-изображение `public/media/hero-poster.jpg`), чтобы не тратить трафик
- [ ] AC-7: Видео не влияет на время загрузки страницы (lazy load, preload="none" или "metadata")
- [ ] AC-8: Дизайн остаётся в рамках DESIGN.md (тёмная тема, Manrope, Framer Blue акценты)

### US-2: Отзывы с Яндекс Карт

- **Как** посетитель сайта (USER)
- **Я хочу** видеть реальные отзывы клиентов с Яндекс Карт
- **Чтобы** убедиться в качестве парка на основе независимых мнений

**Acceptance Criteria:**
- [ ] AC-1: На лендинге появляется секция "Отзывы" между "Преимущества" и "Контакты"
- [ ] AC-2: API route `GET /api/reviews` возвращает список отзывов в стандартном формате `{ success: true, data: [...] }`
- [ ] AC-3: Каждый отзыв показывает: имя автора, рейтинг (звёзды), текст отзыва, дата
- [ ] AC-4: Отзывы отображаются в виде горизонтальной карусели (свайп на мобиле, стрелки на десктопе)
- [ ] AC-5: Отзывы кэшируются на сервере (минимум 1 час) — не парсить Яндекс на каждый запрос
- [ ] AC-6: Если парсинг недоступен — показывается fallback с текстом "300+ отзывов на Яндекс Картах" и ссылкой
- [ ] AC-7: Ссылка "Все отзывы на Яндекс Картах →" ведёт на страницу парка на Яндекс Картах
- [ ] AC-8: Дизайн карточек отзывов — тёмная тема (чёрный фон, белый текст, Framer Blue акценты, pill-shape элементы)
- [ ] AC-9: Минимум 5, максимум 15 последних отзывов на странице

## Приоритет (MoSCoW)

### Must have
- Видео-фон на Hero с overlay и fallback (US-1: AC-1 через AC-6)
- Секция отзывов с карточками (US-2: AC-1, AC-3, AC-4, AC-8)
- API route для отзывов с кэшированием (US-2: AC-2, AC-5)

### Should have
- Fallback при недоступности видео/отзывов (US-1: AC-5, US-2: AC-6)
- Мобильная оптимизация видео — постер вместо видео (US-1: AC-6)
- Ссылка на все отзывы на Яндекс Картах (US-2: AC-7)

### Could have
- Анимация появления карточек отзывов при скролле
- Счётчик общего количества отзывов и среднего рейтинга над каруселью

### Won't have (не в этой итерации)
- Модерация отзывов (показываем все с Яндекса как есть)
- Ответы на отзывы через наш сайт
- Загрузка видео через админку (пока вручную в `public/media/`)

## Метрики успеха

| Метрика | Базовое (сейчас) | Целевое |
|---------|-----------------|---------|
| Время на первом экране | ~3 сек | >6 сек (видео удерживает) |
| Скролл до секции контактов | ~30% посетителей | >50% (отзывы мотивируют) |
| Клики на waitlist CTA | baseline TBD | +20% от текущего |

## Вне скоупа

- Смена структуры остальных секций (Офисы, Услуги, Преимущества — без изменений)
- Загрузка видео через админ-панель
- Интеграция с другими отзовиками (Google, 2GIS)
- Видео на мобильных устройствах (только постер)
- Форма для оставления отзывов на нашем сайте

## Технические заметки (для Architect)

- Ссылку на Яндекс Карты пользователь пришлёт позже — API route должен принимать URL из env-переменной `YANDEX_MAPS_URL`
- Видео файл пользователь загрузит в `public/media/` — компонент должен работать и без видео
- Текущий лендинг: `src/app/page.tsx` → компоненты в `src/components/public/landing/`
- Дизайн-система: `DESIGN.md` (Framer-style, тёмная тема)
