# PRD: Mega-Landing Enhancement для Платформы "Деловой"

## Проблема

Текущий лендинг платформы функционален и имеет правильную структуру (7 секций, адаптивный дизайн, валидация форм), но выглядит **статичным и базовым**. В конкурентной среде бизнес-парков и коворкингов первое впечатление критично — потенциальные арендаторы и клиенты сервисов ожидают **современный, интерактивный, визуально впечатляющий** опыт, который отражает премиальность бизнес-парка "Деловой".

**Боли целевой аудитории:**

### USER (Потенциальный арендатор / клиент)
- **Первое впечатление**: статичная страница не вызывает доверия и не демонстрирует современность бизнес-парка
- **Ориентирование**: непонятно, что предлагает парк, без визуального акцента на ключевых преимуществах
- **Вовлечённость**: отсутствие интерактивности и анимаций → низкое время на сайте → меньше конверсий в waitlist
- **Мобильный опыт**: на смартфоне лендинг работает, но не "оживает" — упущенная возможность

### SUPERADMIN (Владелец бизнес-парка)
- **Конкурентность**: другие бизнес-парки используют современные веб-технологии, наш лендинг отстаёт
- **Конверсия**: низкая конверсия посетителей в заявки на аренду офисов (waitlist)
- **Брендинг**: сайт не отражает премиальность и инновационность парка

**Бизнес-метрики, которые страдают:**
- Конверсия в waitlist: ~1-2% (индустриальный стандарт для премиум-сегмента: 5-8%)
- Bounce rate: >70% (целевой: <50%)
- Время на сайте: <30 секунд (целевой: 1.5-2 минуты)
- Mobile engagement: отсутствие интерактивности снижает конверсию на мобильных на 60%

---

## Решение

Превратить текущий базовый лендинг в **премиальный, современный mega-landing** с Framer-style анимациями, сохраняя существующую структуру и улучшая визуальный опыт и интерактивность.

**Ключевые улучшения:**

1. **Hero Section с "Wow-эффектом"**
   - Scroll-triggered parallax для заголовка и фона
   - Animated counters для статистики (офисы, клиенты, м² площади)
   - Gradient overlays и glow effects вокруг CTA-кнопок
   - Видео-фон или анимированный gradient background

2. **Framer-Style Анимации при скролле**
   - Fade-in + slide-up для всех секций при попадании в viewport
   - Stagger animations для карточек офисов, услуг, преимуществ
   - Smooth scroll anchors для навигации
   - Parallax-эффекты для фоновых элементов

3. **Интерактивные компоненты**
   - Sticky navbar с изменением цвета/прозрачности при скролле
   - Hover-эффекты для карточек с 3D-трансформациями (tilt effect)
   - Animated waitlist form с micro-interactions
   - Toast notifications для успешной отправки заявки

4. **Визуальное обновление**
   - Обновлённая цветовая палитра с акцентом на синий (#0099ff)
   - Glassmorphism эффекты для карточек
   - Gradient borders и box-shadows с glow
   - Оптимизированная типографика с Manrope (уже подключён)

5. **Производительность**
   - Lazy loading для изображений
   - Оптимизация анимаций (60 FPS)
   - Responsive для всех устройств (mobile-first)

**Технологический стек:**
- **Framer Motion** — библиотека анимаций (React-native, декларативный синтаксис)
- **Tailwind CSS v4** — сохраняем текущий подход к стилям
- **Next.js 15 Image** — оптимизация изображений
- **Intersection Observer API** — триггеры для scroll-анимаций

## Целевая аудитория

### Первичная: USER (Потенциальный арендатор / клиент)
- Ищет офис для аренды или узнаёт о сервисах парка
- Первое посещение сайта — критичная точка принятия решения
- Ожидает современный, профессиональный опыт

### Вторичная: SUPERADMIN (Владелец бизнес-парка)
- Нужен инструмент для привлечения арендаторов
- Требуется аналитика по конверсиям (waitlist submissions)
- Хочет гордиться сайтом как витриной бизнеса

## User Stories

### US-1: Hero Section с Wow-эффектом и анимированной статистикой

**Как** посетитель сайта (USER)
**Я хочу** видеть впечатляющий Hero-блок с плавными анимациями и живой статистикой
**Чтобы** сразу понять масштаб и премиальность бизнес-парка "Деловой"

**Acceptance Criteria:**
- [ ] AC-1.1: Hero-секция занимает 100vh на десктопе, адаптируется на мобильных
- [ ] AC-1.2: Заголовок появляется с fade-in + slide-up анимацией при загрузке страницы (duration: 800ms)
- [ ] AC-1.3: Подзаголовок появляется через 200ms после заголовка (stagger effect)
- [ ] AC-1.4: Статистика (кол-во офисов, м², клиентов) отображается с animated counters (от 0 до целевого значения за 2 секунды)
- [ ] AC-1.5: CTA-кнопки ("Выбрать офис", "Записаться в лист ожидания") имеют glow-эффект при hover (box-shadow с #0099ff)
- [ ] AC-1.6: Фоновый gradient анимирован (медленное смещение цветов) ИЛИ видео-фон (если есть контент)
- [ ] AC-1.7: При скролле вниз Hero-элементы уходят с parallax-эффектом (заголовок медленнее фона)
- [ ] AC-1.8: Все анимации работают плавно на 60 FPS (проверено в Chrome DevTools Performance)

---

### US-2: Scroll-Triggered Reveals для всех секций

**Как** посетитель сайта (USER)
**Я хочу** видеть плавное появление контента при скролле вниз по странице
**Чтобы** контент не казался перегруженным и вовлекал меня в изучение сайта

**Acceptance Criteria:**
- [ ] AC-2.1: Каждая секция (Offices, Services, Advantages, Contacts) появляется с fade-in + slide-up при попадании в viewport
- [ ] AC-2.2: Триггер анимации — когда секция на 20% видна в viewport
- [ ] AC-2.3: Анимация срабатывает только один раз (не повторяется при обратном скролле)
- [ ] AC-2.4: Duration анимации: 600ms с easing `ease-out`
- [ ] AC-2.5: Карточки внутри секций (офисы, услуги, преимущества) появляются с stagger-эффектом (каждая следующая с задержкой 100ms)
- [ ] AC-2.6: На мобильных устройствах анимации сохраняются, но stagger delay сокращён до 50ms для быстроты
- [ ] AC-2.7: Если JavaScript отключён, контент отображается статично без анимаций (progressive enhancement)

---

### US-3: Интерактивный Navbar с анимацией при скролле

**Как** посетитель сайта (USER)
**Я хочу** видеть навбар, который адаптируется при скролле и помогает ориентироваться на странице
**Чтобы** легко переходить между секциями и не терять контекст

**Acceptance Criteria:**
- [ ] AC-3.1: При загрузке страницы navbar полностью прозрачный с белым текстом
- [ ] AC-3.2: При скролле вниз на 100px navbar получает backdrop-blur (glassmorphism) и полупрозрачный чёрный фон (rgba(0,0,0,0.8))
- [ ] AC-3.3: Переход между состояниями плавный (transition: 300ms)
- [ ] AC-3.4: Логотип "Деловой" имеет hover-эффект (scale 1.05, transition 200ms)
- [ ] AC-3.5: Навигационные ссылки подсвечиваются при hover (цвет #0099ff, underline animation)
- [ ] AC-3.6: При клике на ссылку navbar (например, "Офисы", "Услуги") выполняется smooth scroll к секции (behavior: smooth)
- [ ] AC-3.7: На мобильных устройствах мобильное меню открывается с slide-in анимацией слева (duration: 300ms)
- [ ] AC-3.8: Navbar фиксирован на всех разрешениях (position: fixed)

### US-4: Карточки офисов с hover-эффектами и улучшенной формой Waitlist

**Как** потенциальный арендатор (USER)
**Я хочу** видеть офисы с интерактивными карточками и удобную форму записи в лист ожидания
**Чтобы** быстро оценить варианты и оставить заявку без лишних кликов

**Acceptance Criteria:**
- [ ] AC-4.1: Каждая карточка офиса при hover поднимается вверх на 8px (translateY) и получает увеличенную тень
- [ ] AC-4.2: Hover-эффект включает лёгкий 3D-tilt (перспектива при движении мыши) — опционально для "wow-эффекта"
- [ ] AC-4.3: Badge "Sold Out" имеет пульсирующий glow-эффект (красный цвет, animate-pulse от Tailwind)
- [ ] AC-4.4: Waitlist-форма отображается под карточками офисов
- [ ] AC-4.5: Поля формы (Имя, Телефон) имеют focus-состояние с синей рамкой (#0099ff)
- [ ] AC-4.6: При клике на "Отправить" кнопка показывает loading spinner (анимация вращения)
- [ ] AC-4.7: После успешной отправки появляется toast notification с зелёной галочкой и текстом "Заявка принята!" (автоматически исчезает через 4 секунды)
- [ ] AC-4.8: При ошибке отправки — toast notification с красным крестиком и текстом "Ошибка. Попробуйте позже."
- [ ] AC-4.9: Toast notifications появляются с slide-in анимацией сверху (top-right corner)
- [ ] AC-4.10: Форма валидируется на клиенте (Zod-схема), ошибки отображаются под полями красным текстом

## Приоритет (MoSCoW)

### Must Have
**Категория:** Must have
**Обоснование:** Без этих элементов лендинг останется базовым и не достигнет цели "мега-лендинга". Эти фичи напрямую влияют на конверсию в waitlist и первое впечатление.

1. **US-1** — Hero Section с анимациями и counters (главный wow-эффект)
2. **US-2** — Scroll-triggered reveals для секций (основа Framer-style опыта)
3. **US-3** — Sticky navbar с анимацией (навигация и UX)
4. **US-4** — Hover-эффекты для карточек офисов и waitlist form с toast notifications (конверсия)
5. **US-8** — Производительность и адаптивность (критично для мобильных пользователей)

**Зависимости:**
- Установка Framer Motion (`npm install framer-motion`)
- Оптимизация изображений (если есть реальные фото)
- Настройка Яндекс.Карт API (для секции контактов)

---

### Should Have
**Категория:** Should have
**Обоснование:** Улучшают визуальный опыт и детализацию, но можно запустить MVP без них.

1. **US-5** — Визуальные акценты для услуг и преимуществ (улучшает презентацию, но не критично)
2. **US-6** — Интерактивная карта и улучшенные контакты (важно для навигации, но можно сделать базовую версию)
3. **US-7** — Footer с брендингом (низкая приоритетность, но завершает дизайн)

---

### Could Have
**Категория:** Could have
**Обоснование:** Дополнительные "wow-эффекты", которые можно добавить в Phase 2 после запуска основного функционала.

1. **3D Tilt Effect** для карточек (AC-4.2) — опциональный hover-эффект
2. **Parallax для Hero Background** (AC-1.7) — дополнительный визуальный эффект
3. **Animated Gradient Background** в Hero (AC-1.6) — если есть время на fine-tuning
4. **Дополнительные секции:**
   - Testimonials (отзывы клиентов) — если есть контент
   - FAQ (часто задаваемые вопросы) — улучшает SEO
   - Timeline (история парка) — сторителлинг

---

### Won't Have (в этой итерации)
**Категория:** Won't have
**Обоснование:** Осознанно откладываем, чтобы сфокусироваться на запуске MVP мега-лендинга.

1. **Мобильное приложение** — не входит в скоуп лендинга
2. **Интеграция с CRM** — форма waitlist уже отправляет данные в API, дополнительная интеграция не требуется
3. **Multilanguage (EN version)** — сначала запускаем на русском
4. **Dark/Light mode toggle** — дизайн изначально тёмный, переключатель избыточен
5. **Video Background в Hero** — требует наличия профессионального видео-контента (если его нет)
6. **Advanced 3D эффекты** (WebGL, Three.js) — слишком сложно для первой итерации
7. **Chatbot / Live Chat** — отдельная интеграция, не часть лендинга
8. **Blog Section** — контентная стратегия вне скоупа лендинга

## Метрики успеха

### Baseline (текущее состояние)
- **Конверсия в waitlist:** 1-2% (оценка)
- **Bounce rate:** 70%+
- **Время на сайте:** <30 секунд
- **Mobile traffic:** ~40% от всех посетителей
- **Mobile conversion:** 0.5-1% (в 2 раза ниже десктопа)

### Target (через 4 недели после запуска)
- **Конверсия в waitlist:** 5-8% (+250-400%)
- **Bounce rate:** <50% (-30%)
- **Время на сайте:** 1.5-2 минуты (+200%)
- **Mobile conversion:** 3-5% (приближается к десктопу)
- **Scroll Depth:** 60%+ пользователей доскролливают до секции "Контакты"

### Измерение
- **Google Analytics 4** (или Yandex.Metrica) — события:
  - `hero_cta_click` — клик на CTA в Hero
  - `waitlist_submit` — отправка формы waitlist
  - `section_view` — просмотр секций (через Intersection Observer)
  - `office_card_hover` — взаимодействие с карточками офисов
  - `navbar_link_click` — клики по навигации

- **Web Vitals** (Lighthouse CI):
  - LCP < 2.5s (десктоп), < 4s (мобильные)
  - FID < 100ms
  - CLS < 0.1

- **A/B тестирование** (опционально):
  - Версия A: текущий лендинг (baseline)
  - Версия B: мега-лендинг с анимациями
  - Сравнение конверсий через 2 недели

## Вне скоупа

Что **НЕ** входит в эту итерацию (но может быть в будущих фазах):

1. **Новые модули или функционал** — только визуальное улучшение существующего лендинга, никаких изменений в API или бэкенде
2. **Админ-панель для редактирования лендинга** — контент хардкодится в компонентах, CMS не требуется
3. **Интеграция с платёжными системами** — форма waitlist только собирает контакты, оплата не нужна
4. **Email-маркетинг** — после отправки формы пользователь НЕ получает автоматические письма (это отдельная задача для MANAGER)
5. **SEO-оптимизация** — мета-теги и Open Graph уже настроены в layout.tsx, дополнительная SEO-работа вне скоупа
6. **Accessibility (A11y) audit** — базовая доступность (keyboard navigation, alt-теги) обязательна, но полноценный WCAG audit откладывается
7. **Переписывание существующих модулей** (cafe, ps-park, gazebos) — они остаются как есть, улучшаем только главную страницу
8. **Интеграция с соцсетями** (автопостинг, виджеты) — ссылки в Footer достаточно
9. **Analytics Dashboard для SUPERADMIN** — аналитика лендинга смотрится в Google Analytics, встроенный дашборд не требуется

## Дизайн-референсы

**Стиль:** Framer.com, Linear.app, Vercel.com — чистый, минималистичный, с акцентом на типографику и анимации.

**Цветовая палитра (сохраняем текущую + доработки):**
- **Background:** `#000000` (чёрный)
- **Text Primary:** `#FFFFFF` (белый)
- **Text Secondary:** `#A6A6A6` (серый)
- **Accent:** `#0099FF` (синий) — используется для CTA, hover, glow
- **Accent Gradient:** `linear-gradient(135deg, #0099FF 0%, #00D4FF 100%)`
- **Error:** `#FF4444` (красный)
- **Success:** `#00FF88` (зелёный)

**Типографика (уже настроена):**
- **Headings:** Manrope (800, 700) — крупные заголовки, Hero
- **Subheadings:** Manrope (600, 500) — подзаголовки секций
- **Body:** Inter (400) — описания, тексты

**Анимации:**
- **Duration:** 200ms (micro), 300-600ms (transitions), 800ms+ (entrances)
- **Easing:** `ease-out` (default), `cubic-bezier(0.4, 0, 0.2, 1)` (custom)

## Связь с CLAUDE.md

Эта фича полностью соответствует принципам из `CLAUDE.md`:

1. **API-First** — не нарушается, лендинг остаётся фронтенд-слоем над существующим API `/api/waitlist`
2. **Progressive Enhancement** — анимации работают через JavaScript, но при его отключении контент остаётся доступным
3. **Domain Modules** — лендинг не затрагивает модули (cafe, ps-park, gazebos), только главную страницу
4. **Тестирование** — для каждого компонента пишутся тесты (Vitest), проверяющие корректность анимаций и интерактивности
5. **Git Workflow** — работа ведётся в ветке `claude/mega-landing-enhancement`, PR перед мержем в `main`
