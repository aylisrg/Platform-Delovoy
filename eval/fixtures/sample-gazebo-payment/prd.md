# PRD: Онлайн-оплата бронирования беседки

## Проблема
Сейчас пользователи бронируют беседки без предоплаты. Менеджер вручную подтверждает брони по телефону — это долго и часто теряем 30% броней из-за неудобного процесса.

## Решение
Добавить онлайн-оплату при создании брони. После оплаты бронь автоматически переходит в статус CONFIRMED.

## Целевая аудитория
USER (посетитель парка)

## User Stories

### US-1: Оплата при создании брони
- **Как** посетитель парка
- **Я хочу** оплатить бронь онлайн сразу после выбора даты
- **Чтобы** не ехать в парк для предоплаты

**Acceptance Criteria:**
- [ ] AC-1: После выбора даты появляется кнопка "Оплатить"
- [ ] AC-2: После успешной оплаты статус брони = CONFIRMED
- [ ] AC-3: Пользователь получает подтверждение в Telegram и email
- [ ] AC-4: При отмене < 24ч — возврат 50%, иначе 100%

### US-2: Менеджер видит оплаты
- **Как** менеджер беседок
- **Я хочу** видеть какие брони оплачены
- **Чтобы** не тратить время на звонки для подтверждения

**Acceptance Criteria:**
- [ ] AC-5: В списке броней помечены оплаченные (бейдж "Paid")
- [ ] AC-6: Менеджер может инициировать возврат одним кликом

## Приоритет (MoSCoW)
Must have — без онлайн-оплаты менеджер тратит 2ч/день на звонки, и теряем бронь.

## Метрики успеха
- Базовое: 0% броней оплачены онлайн
- Целевое: 60% через 4 недели после релиза

## Вне скоупа
- Частичная оплата
- Рассрочка
- Оплата через СБП (отдельная фича)
